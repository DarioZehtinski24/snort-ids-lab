# Evidence Explanation

## Purpose
This file explains the main alerts observed in the JSON evidence file generated by Snort during validation of the detection lab.

## Evidence file
The main raw evidence file is:

text
final_detection_evidence.json

This file contains JSON-formatted Snort alerts generated during controlled testing from a second machine in the same network.

Main observed alerts

1. SMB probe detection

Example alert:
{ "timestamp" : "03/25-10:31:01.982890", "proto" : "TCP", "src_ap" : "192.168.0.147:56224", "dst_ap" : "192.168.0.101:445", "rule" : "1:1000003:1", "msg" : "LOCAL SMB Probe Detected", "action" : "allow" }

Explanation:

192.168.0.147 is the test machine
192.168.0.101:445 is the monitored host and SMB port
rule 1000003 is the custom SMB detection rule
the alert confirms that the SMB probe rule was successfully triggered

2. RDP probe detection

Example alert:
{ "timestamp" : "03/25-10:31:01.978938", "proto" : "TCP", "src_ap" : "192.168.0.147:56224", "dst_ap" : "192.168.0.101:3389", "rule" : "1:1000004:1", "msg" : "LOCAL RDP Probe Detected", "action" : "allow" }

Explanation:

the test machine sent a TCP probe to port 3389
rule 1000004 is the custom RDP detection rule
the alert confirms successful detection of an RDP probe

3. Repeated SSH connection attempts

Example alert:
{ "timestamp" : "03/25-10:35:13.811707", "proto" : "TCP", "src_ap" : "192.168.0.147:47770", "dst_ap" : "192.168.0.101:22", "rule" : "1:1000001:2", "msg" : "LOCAL Repeated SSH Connection Attempts", "action" : "allow" }

Explanation:

the test machine generated repeated connection attempts toward port 22
rule 1000001 is the custom SSH detection rule
the rule was configured with threshold logic and triggered successfully

4. Built-in TCP port scan detection

Example alert:
{ "timestamp" : "03/25-10:31:01.981375", "proto" : "TCP", "src_ap" : "192.168.0.147:56224", "dst_ap" : "192.168.0.101:5900", "rule" : "122:1:1", "msg" : "(port_scan) TCP portscan", "action" : "allow" }

Explanation:

this alert was generated by Snort’s built-in port_scan inspector
it indicates scanning behavior from the test machine toward the monitored host
this confirms that built-in scan detection was active and functioning
Interpretation

The evidence file confirms that the Snort detection lab successfully detected:

custom SMB probe attempts
custom RDP probe attempts
repeated SSH connection attempts
built-in TCP port scan activity
Note

Some additional background or decoder-related alerts may appear in raw evidence depending on the current Snort configuration and live network traffic. The most relevant alerts for project validation are the custom alerts and the built-in TCP port scan alert described above.
