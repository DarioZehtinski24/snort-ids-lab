# Evidence Explanation

## Purpose

This file explains the main alerts observed in the JSON evidence file generated by Snort during validation of the detection lab.

---

## Evidence File

The main raw evidence file is:

```text
final_detection_evidence.json
```

This file contains JSON-formatted Snort alerts generated during controlled testing from a second machine within the same network.

---

## Main Observed Alerts

### SMB Probe Detection

#### Example Alert

```json
{
  "timestamp": "03/25-10:31:01.982890",
  "proto": "TCP",
  "src_ap": "192.168.0.147:56224",
  "dst_ap": "192.168.0.101:445",
  "rule": "1:1000003:1",
  "msg": "LOCAL SMB Probe Detected",
  "action": "allow"
}
```

#### Explanation

- `192.168.0.147` is the test machine
- `192.168.0.101:445` is the monitored host and SMB port
- `rule 1000003` is the custom SMB detection rule

This alert confirms that the SMB probe rule was successfully triggered.

### Built-in TCP Port Scan Detection

#### Example Alert

```json
{
  "timestamp": "03/25-10:31:01.981375",
  "proto": "TCP",
  "src_ap": "192.168.0.147:56224",
  "dst_ap": "192.168.0.101:5900",

  "action": "allow"

```

#### Explanation

This alert was generated by Snort’s built-in `port_scan` inspector.

It indicates scanning behavior from the test machine toward the monitored host.

This confirms that built-in scan detection was active and functioning correctly.

The evidence file confirms that the Snort detection lab successfully detected:

- custom SMB probe attempts
- custom RDP probe attempts
- repeated SSH connection attempts
- built-in TCP port scan activity

This validates both the custom rules and the built-in Snort detection mechanisms.
